القسم ده بيتكلم عن **كيفية التنقل بين أوضاع الأوامر في نظام Cisco IOS**، وهو النظام اللي بيستخدم لإدارة أجهزة الشبكات زي الروترات والسويتشات. هنا الشرح بطريقة مبسطة:

---

#### 1. **أوضاع الأوامر الرئيسية (Primary Command Modes):**
في Cisco IOS، فيه وضعين رئيسيين للوصول للأوامر:

- **User EXEC Mode:**
  - ده الوضع الأساسي اللي بيدخله المستخدم أول ما يتصل بالجهاز.
  - بيسمح بأوامر محدودة جدًا (زي مراقبة بسيطة للجهاز).
  - العلامة: الـ prompt بينتهي بـ **>** (مثال: `Switch>`).

- **Privileged EXEC Mode:**
  - ده الوضع المتقدم اللي بيسمح بالوصول لكل الأوامر والمميزات (زي التحكم الكامل بالجهاز).
  - العلامة: الـ prompt بينتهي بـ **#** (مثال: `Switch#`).

---

### ملخص باختصار:
- **User EXEC Mode**: أوامر محدودة، الـ prompt بيكون `>`.
- **Privileged EXEC Mode**: التحكم الكامل، الـ prompt بيكون `#`.
- **Global Config Mode**: إعدادات عامة للجهاز.
- **Line Config Mode**: إعدادات الوصول (Console، SSH، Telnet).
- **Interface Config Mode**: إعدادات المنافذ أو الواجهات.
- **التنقل**:
  - `enable`: من User EXEC لـ Privileged EXEC.
  - `configure terminal`: لـ Global Config.
  - `line` أو `interface`: لأوضاع التهيئة الفرعية.
  - `exit`: رجوع خطوة للخلف.
  - `end` أو `Ctrl + Z`: رجوع لـ Privileged EXEC.

لو عايز توضيح أكتر عن أي جزئية أو أمثلة عملية، قولي!
