#### 1. **هيكلية الأوامر في Cisco IOS (Basic IOS Command Structure):**
- كل أمر في Cisco IOS بيتكون من جزئين:
  - **Keyword (كلمة مفتاحية):** دي كلمة محددة مسبقًا في النظام (زي `ip protocols`).
  - **Argument (القيمة):** دي قيمة أو متغير بتدخله أنت بنفسك (زي عنوان IP، مثلاً `192.168.10.5`).
  - مثال: الأمر `ping 192σι.168.10.5`:
    - `ping` هو الكلمة المفتاحية (Keyword).
    - `192.168.10.5` هو القيمة اللي بتدخلها (Argument).

### ملخص باختصار:
- **هيكلية الأوامر:** الأوامر بتتكون من **Keywords** (ثابتة) و **Arguments** (بتدخلها أنت).
- **فحص الصيغة:** النظام بيوضح صيغة الأوامر (Bold للثابت، Italics للمتغير، [ ] للاختياري، { } للإجباري).
- **المساعدة:**
  - **Context-Sensitive Help**: اكتب `?` عشان تعرف الأوامر والخيارات.
  - **Syntax Check**: النظام بيبلغك لو الأمر غلط.
- **اختصارات:**
  - **Tab**: يكمل الأمر.
  - **Ctrl+C / Ctrl+Z**: يرجعك لـ Privileged EXEC.
  - **Ctrl+Shift+6**: يوقف العمليات.
  - **Up Arrow**: يرجع الأوامر السابقة.
  - لو الناتج طويل (`--More--`): **Enter** للسطر التالي، **Spacebar** للصفحة التالية.
- **تطبيقات عملية:** بتساعدك تتدرب على الاتصال بالأجهزة، استخدام CLI، وتهيئة إعدادات بسيطة.
