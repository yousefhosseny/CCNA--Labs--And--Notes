### شرح القسم عن "IP Vulnerabilities and Threats" (ثغرات وتهديدات IP) بأبسط طريقة ممكنة

هذا القسم من كورس سيسكو عن الأمن السيبراني، يتكلم عن الثغرات في بروتوكول IP (IPv4 وIPv6) وكيف يستغلها الهكرز في هجمات. IP مش بيحقق من مصدر الحزمة، فالهكر يقدر يزيف العناوين. هقسمه لك عشان يكون سهل، مع جداول للأنواع:

#### 1. **فيديو عن هجمات IP وICMP الشائعة (Video – Common IP and ICMP Attacks)**
   - الفيديو يغطي:
     - تقنيات هجمات IP: ICMP، تضخيم وانعكاس (amplification and reflection)، تزييف العناوين، رجل في الوسط (MITM)، سرقة الجلسة (session hijacking).
     - تقنيات هجمات ICMP: طلب/رد الصدى (echo)، غير قابل للوصول (unreachable)، رد القناع (mask reply)، إعادة توجيه (redirects)، اكتشاف الراوتر (router discovery).

#### 2. **IPv4 وIPv6 (IPv4 and IPv6)**
   - IP مش بيحقق إذا عنوان المصدر صحيح، فالهكر يقدر يزيف العنوان (spoofing).
   - أنواع الهجمات الشائعة المتعلقة بـIP (من الجدول):
     | نوع الهجوم | الوصف |
     |------------|--------|
     | **ICMP attacks** | يستخدم ICMP (زي البينج) عشان يكتشف الشبكات، يعمل DoS، أو يغير جداول التوجيه. |
     | **Amplification and reflection attacks** | يعمل DoS أو DDoS عشان يمنع المستخدمين الشرعيين من الوصول. |
     | **Address spoofing attacks** | يزيف عنوان IP عشان يخفي هويته أو يتظاهر بجهاز آخر (blind أو non-blind). |
     | **Man-in-the-Middle (MITM)** | يقف بين المرسل والمستقبل، يراقب، يسرق، أو يغير البيانات بدون ما حد يعرف. |
     | **Session hijacking** | يسرق جلسة مفتوحة باستخدام MITM، بعد دخول الشبكة. |

#### 3. **هجمات ICMP (ICMP Attacks)**
   - ICMP يستخدم في الاستطلاع (reconnaissance) والمسح: يكتشف الشبكة، الأجهزة الفعالة، نوع النظام، حالة الجدار الناري، أو يعمل DoS.
   - ICMPv4 وICMPv6 عرضة لنفس الهجمات.
   - الحل: استخدم فلاتر ACL صارمة على حواف الشبكة، وأجهزة أمن زي firewalls وIDS عشان تكشفها.
   - رسائل ICMP اللي يستخدمها الهكرز (من الجدول):
     | الرسالة | الوصف |
     |---------|--------|
     | **ICMP echo request and echo reply** | يتحقق من وجود الجهاز أو يعمل DoS. |
     | **ICMP unreachable** | يمسح الشبكة ويكتشفها. |
     | **ICMP mask reply** | يرسم خريطة الشبكة الداخلية. |
     | **ICMP redirects** | يخدع الجهاز عشان يبعت حركته عبر جهاز مصاب، يعمل MITM. |
     | **ICMP router discovery** | يضيف مسارات مزيفة في جدول التوجيه. |

#### 4. **فيديو عن التضخيم، الانعكاس، والتزييف (Video – Amplification, Reflection, and Spoofing Attacks)**
   - الفيديو يشرح هجمات التضخيم (amplification)، الانعكاس (reflection)، والتزييف (spoofing).

#### 5. **هجمات التضخيم والانعكاس (Amplification and Reflection Attacks)**
   - يستخدمها الهكر عشان يعمل DoS: يضخم الحركة أو ينعكسها على الضحية.
   - مثال: هجوم Smurf، يبعت بينج مزيف لشبكة كاملة، الجميع يرد على الضحية ويغرقها.
   - أشكال حديثة: DNS أو NTP amplification.
   - كمان هجمات استنزاف الموارد: يستهلك موارد الشبكة أو الجهاز عشان يتعطل.

#### 6. **هجمات تزييف العناوين (Address Spoofing Attacks)**
   - الهكر يصنع حزم بـIP مصدر مزيف عشان يخفي هويته أو يتظاهر بمستخدم شرعي. غالباً مع هجمات أخرى زي Smurf.
   - أنواع:
     - **Non-blind spoofing**: الهكر يشوف الحركة بين الجهاز والضحية، يحدد حالة الجدار الناري، يتنبأ بالأرقام، أو يسرق جلسة.
     - **Blind spoofing**: مش يشوف الحركة، يستخدم في DoS.
   - **MAC address spoofing**: داخل الشبكة، يغير عنوان MAC لجهازه عشان يطابق جهاز آخر.

في النهاية، القسم ده يركز على إن IP ضعيف في التحقق، فالحماية تحتاج فلاتر، firewalls، ومراقبة. لو عايز تفاصيل أكتر عن فيديو أو نوع هجوم، قولي!
