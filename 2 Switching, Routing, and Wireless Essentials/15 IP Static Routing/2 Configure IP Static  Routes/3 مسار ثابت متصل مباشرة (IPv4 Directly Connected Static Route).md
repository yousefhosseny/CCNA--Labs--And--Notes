
### 3. **مسار ثابت متصل مباشرة (IPv4 Directly Connected Static Route)**
- هنا بتحدد بس **واجهة الخروج (Exit Interface)** بدل عنوان الـ Next-Hop.
- **ملاحظة**: الطريقة دي بتُستخدم بس مع واجهات تسلسلية نقطة-لنقطة (Point-to-Point Serial Interfaces)، ويفضل دايمًا استخدام عنوان الـ Next-Hop.
- **مثال على R1**:
  ```bash
  ip route 172.16.1.0 255.255.255.0 s0/1/0
  ip route 192.168.1.0 255.255.255.0 s0/1/0
  ip route 192.168.2.0 255.255.255.0 s0/1/0
  ```
- **التفسير**:
  - بتحدد الشبكة الوجهة (مثل 172.16.1.0/24) وواجهة الخروج (Serial0/1/0).
  - الحزم اللي رايحة للشبكات دي هتترسل من واجهة s0/1/0.
