### 3. **حل مشكلة الاتصال (Solve a Connectivity Problem)**
- **المشكلة**: الاتصال من PC1 لـ PC3 فشل.
- **خطوات التشخيص**:
  1. **اختبار بـ Ping**:
     - من واجهة R1 (G0/0/0) لـ PC3: فشل.
     - من واجهة R1 (S0/1/0) لـ R2: نجح.
     - من واجهة R1 (S0/1/0) لـ R3: نجح.
     - يعني المشكلة مش في الاتصال بين R1 وR2 أو R1 وR3، لكن في مسار لشبكة PC3.
  2. **فحص جدول التوجيه في R2**:
     - الأمر:
       ```bash
       show ip route | begin Gateway
       ```
     - المخرجات بتوري:
       - `C 172.16.1.0/24`: شبكة متصلة مباشرة (GigabitEthernet0/0/0).
       - `L 172.16.1.1/32`: عنوان محلي.
       - `C 172.16.2.0/24`: شبكة متصلة مباشرة (Serial0/1/0).
       - `L 172.16.2.2/32`: عنوان محلي.
       - `S 172.16.3.0/24 [1/0] via 192.168.1.1`: مسار ثابت غلط لشبكة PC3.
       - `C 192.168.1.0/24`: شبكة متصلة مباشرة (Serial0/1/1).
       - `L 192.168.1.2/32`: عنوان محلي.
       - `S 192.168.2.0/24 [1/0] via 192.168.1.1`: مسار ثابت لشبكة أخرى.
  3. **اكتشاف المشكلة**:
     - المسار الثابت في R2 لشبكة 172.16.3.0/24 بيوجه الحزم لعنوان غلط (192.168.1.1).
  4. **حل المشكلة**:
     - إزالة المسار الثابت الغلط (باستخدام `no ip route ...`).
     - إضافة مسار ثابت صحيح:
       ```bash
       ip route 172.16.3.0 255.255.255.0 172.16.2.1
       ```
       - ده بيوجه الحزم لشبكة 172.16.3.0/24 عبر R1 (172.16.2.1).
